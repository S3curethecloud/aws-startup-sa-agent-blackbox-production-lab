# Lab Phase 03: Production Hardening

## Objective

Turn a demo into a production candidate.

## Tasks

1. Add WAF in count mode.
2. Add structured JSON logs and correlation IDs.
3. Add alarms for API errors, SQS backlog, Step Functions failure, Bedrock errors, and cost anomalies.
4. Enable CloudTrail, GuardDuty, and Security Hub in the target account.
5. Review IAM with least privilege.
6. Test backup and restore.
7. Run an incident game day.
8. Update compliance evidence package.

## Acceptance criteria

- Release process is documented.
- Incident runbook is tested.
- Cost dashboard is reviewed.
- Security findings are triaged.
- No production enforcement is enabled without separate approval.
